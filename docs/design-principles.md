# Design principles

The site should feel: detail-oriented, minimal, quietly playful, a little nerdy, and disciplined about process. In practice:

- **Detail over addition** — refine what's already on the page before adding something new.
- **Restraint** — one accent color, minimal typographic roles, few moving parts. If it doesn't clearly serve the feel above, leave it out.
- **Playful, not cute** — motion and interaction should feel like a natural response to input, not a performance.
- **Precision as personality** — the nerdiness shows up in exactness (alignment, timing, consistency), not in imagery or copy.
- **Systems over one-offs** — content stays data-driven so structure itself reflects a designed process.

Weigh new ideas against these before the technical rules in the other docs.

## Do not

- Add Tailwind
- Invent a second palette
- Use purple/AI-slop styling
- Add project detail pages unless asked
- Use squash/squish "jelly" animations
- Add colored glow/bloom shadows
